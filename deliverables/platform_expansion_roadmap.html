3遇到的困难（4 类）

不是所有困难都是技术。法律/合规/商业阻塞往往更难绕。

① 技术反爬（5 级）

难度典型代表反爬技术破解方案
1 · 容易 Article / Overstock / Opiniones UA 检测 / 简单 rate limit curl_cffi + UA 池
2 · 中低 West Elm / Idealo / TrustedShops Cloudflare 基础挑战 Scrapling StealthyFetcher（已就绪）
3 · 中 Wayfair / Houzz / Otto / Bol / CDiscount / Allegro Cloudflare 高级 / Datadome Scrapling + 商业住宅代理 + per-site profile
4 · 难 Walmart / Home Depot / Lowes / Target PerimeterX / Akamai / Geo-block 商业代理 + Scrapling + 慢速 + 失败重试 + 偶尔 Captcha solver
5 · 最难 Amazon × 7 国 PerimeterX + 自家 WAF + Behavior 检测 + Captcha 商业 SaaS（BrightData / ScraperAPI）或 自部署 PX bypass cluster · 成本/工期最大

② 法律 / 合规阻塞

Amazon ToS 风险：Amazon 用户协议明令禁止 scraping，hiQ vs LinkedIn (2022) 判 public data 可抓，但 Amazon 数据是登录可见，灰色。建议路径：
  • 不抓登录后内容（only public PDP / search results）
  • 低频 + 商业代理 + UA 池（降低被诉风险）
  • 合同明示「数据用途为竞品分析，不二次出售」
  • 客户签 NDA + ToS 责任转移条款

③ 商业 / 谈判阻塞

  • 评论 SaaS（Yotpo/Bazaarvoice）：要找到商家 widget key，需客户提供其商家 widget URL。客户不给就抓不全
  • Houzz 登录墙：大部分商品/价格要登录才看，需要客户买一个普通 Pro 账号给我们用
  • Vidaxl Dropshipping API：申请已发 5 天没回，需要客户名义 follow up
  • Wayfair Trade Program：需要美国公司主体 + EIN 才能注册

④ 数据维度阻塞

  • 真实销量：除 Amazon BSR、Wayfair best-seller，其他站没公开销量 → 我们用评论增量倒推估销（已实现）
  • 多语种 NLP：14 国语言 = 14 个 fine-tune 模型？我们用 claude-haiku-4-5 多语种统一处理（已实现）
  • 视频内容 VOC：YouTube 开箱视频是高价值数据，但需 ASR pipeline（Whisper Large v3），算力成本估 $30k/年

6真心话 · 3 件事必须告诉客户

  1. Amazon 不在 Q2 路径：Wave 3，需要 2-3 个月 + 法律咨询 + 8 人周。如果遨森必须要 Amazon 数据，建议 buy BrightData / ScraperAPI 套餐 ~$2000/月，转手价 ~¥30k/月）
