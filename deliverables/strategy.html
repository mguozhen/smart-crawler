② Biore 报告 = 什么需求

这份报告需要的数据维度

读完 Biore 报告，提炼出 5 个核心数据/能力诉求——和我们现有电商竞品数据是正交平行的，需要新建数据层。

📱 多平台采集

TikTok / Instagram 为主，可能延伸到小红书 / 微博 / X / YouTube；hashtag + 帖子 + 评论一体

🏷️ 多维内容标签

每条帖子 5 维标签：主题 / 形式 / 来源 / 市场 / 互动等级。AI 自动分类

😊 情感聚类

7,176 评论 → 聚成 ~30 个语义簇（"lightweight" 387 次/"white cast" 52 次）+ 多语种

⭐ KOL 发现 + 排名

按平台×粉丝×互动综合排名，Brand/Mega/Macro/Micro 分层。可类比 Reddit top_contributors

📊 平台对比 + 时序

TikTok vs IG 互动 13× 差异；180 天滚动窗口；爆款/衰减/新兴 hashtag 趋势

📄 报告生成

最终交付 = HTML 报告（多语言切换 / 侧栏导航 / 图表）。已有 voc 模板可复用

③ 开源底座 audit · GitHub Top 5

能直接拿来用的开源项目

④ 战略方向抽卡 · 必须先选

4 个产品定位选择，影响所有后续设计

把 Biore 社媒需求"接进来"有 4 种完全不同的产品定位。每个方向的客户、定价、技术架构、MVP 范围都不一样。客户决策成本最高的就是这一步。

方向 A · 平稳延伸
🛒

电商品牌情报中心

smart-crawler stays e-commerce-first
社媒只作为"延伸感知层"
核心定位
已有的 46 站 + Trustpilot + Amazon VOC + Reddit 是主线，社媒只补 TikTok/IG 关键 hashtag 监听，回答客户"你的品牌在社媒被讨论得多吗"。
客户
现有客户：Aosom / 跨境电商品牌。社媒是锦上添花，不是核心买点。
MVP 范围
2 周——只加 TikTok + IG 两个 source，每个站点配置 N 个 hashtag，每天采集 → 进 voc_summary 同一个 dashboard。
投入
2 周
客单价
$0（含在现有 SaaS）
风险
低

方向 C · Agent 原生
🤖

Agent-first 社媒 MCP 工具集

不做 UI，只暴露 MCP tools
跟现有 Reddit MCP 同一套路

AI Agent 开发者 / 数据团队 / 已有 BI 的企业。低门槛接入，按 API 调用计费。
MVP 范围
3-4 周——5 个 MCP tools 上线 + 一个示例 notebook 演示「输入品牌名 → Agent 自动生成 Biore-style 报告」。
