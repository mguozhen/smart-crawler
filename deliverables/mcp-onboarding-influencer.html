红人发现 MCP · 团队接入指南

smart-crawler 暴露的 Influencer Discovery MCP 工具 · 替代 Apify + ScraperAPI · 内部团队使用

1. 端点

https://smartcrawler.io/mcp/

HTTP Streamable MCP transport（fastmcp 实现）。所有请求必须带 API Key。

2. 拿 API Key

  1. 登录 smartcrawler 看板：https://smartcrawler.io
  2. 进 API 接入 页面 → 生成新 Key
  3. 得到形如 sck_xxxxxxxxxxxxxxxx 的 token，只显示一次，立即复制保存
  4. 每个团队成员各自生成一个，方便审计 / 单独撤销

3. Claude Desktop 配置

编辑 ~/Library/Application Support/Claude/claude_desktop_config.json（macOS），加 mcpServers 块：

{
  "mcpServers": {
    "smartcrawler": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://smartcrawler.io/mcp/",
        "--header",
        "Authorization: Bearer sck_PASTE_YOUR_KEY_HERE"
      ]
    }
  }
}

保存后完全退出 Claude Desktop 再打开（Cmd+Q，不是关窗口）。聊天框左下角应该出现 🔌 图标，点开能看到 smartcrawler 的 16 个工具。

4. Cursor / Windsurf / Cline 配置

同样格式，路径换成各自的 MCP 配置文件：

客户端配置路径
Cursor~/.cursor/mcp.json 或项目内 .cursor/mcp.json
Windsurf~/.codeium/windsurf/mcp_config.json
Cline (VS Code)VS Code Settings → Cline → MCP Servers
Continue.dev~/.continue/config.json 的 mcpServers 段

5. 红人发现 4 个新工具

discover_creators_by_hashtag 替代 Apify

按 hashtag 发现 TikTok / Instagram / Facebook 创作者。

参数：

  • platform: "tiktok" / "instagram" / "facebook"
  • hashtags: 字符串数组，不带 #，如 ["amazonfba", "amazonseller"]。Facebook 平台下当搜索关键词用。
  • limit: 1-200，默认 38

返回：

{
  "platform": "instagram",
  "hashtags": ["amazonfba"],
  "itemCount": 12,
  "items": [
    {
      "channelId": "ig:sellerjoe",
      "name": "Seller Joe",
      "platform": "Instagram",
      "profileUrl": "https://www.instagram.com/sellerjoe/",
      "handle": "sellerjoe",
      "followerCount": 12345,
      "email": "biz@sellerjoe.com",
      "websiteUrl": "https://sellerjoe.com"
    }
    ...
  ]
}

enrich_youtube_about 替代 ScraperAPI

从 YouTube 频道 About 页抽取邮箱 + 外链。

参数：urls: YouTube 频道 URL 列表

返回：

{
  "itemCount": 2,
  "items": [
    {"url": "https://www.youtube.com/@MrBeast",
     "email": null, "websiteUrl": "https://bit.ly/mrbeastbowandarrow"},
    {"url": "https://www.youtube.com/@MKBHD/about",
     "email": null, "websiteUrl": "https://www.shopmkbhd.com"}
  ]
}

ingest_tiktok_phone_results 手机驱动专用

给 matrix-mvp 手机驱动调用 —— 接收从安卓 TikTok app 解析出来的创作者批次。

参数：hashtag: 字符串，items: [{authorMeta: {...}}, ...]（Apify 兼容形态）

返回：{runId, datasetId, itemCount} —— 后续用 get_discover_run_items 取数据

get_discover_run_items

取一个 discover run 已采集到的 items（带 limit/offset 分页）。runs 在内存里保留 1 小时后 GC。

8. 验证接入是否成功

Claude Desktop / Cursor 配好后，在聊天里直接说：

用 smartcrawler 帮我抓 instagram 上 hashtag amazonfba 的前 20 个创作者

客户端会触发 discover_creators_by_hashtag(platform="instagram", hashtags=["amazonfba"], limit=20) 并把 JSON 结果展示出来。如果看到 cookies_expired_instagram，说明工具通了但运维还没丢 cookie 文件。

命令行直测（替换 sck_...）：

curl -sS -X POST https://smartcrawler.io/mcp/ \
  -H "Authorization: Bearer sck_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":1}'

速率限制 / 注意事项

  • 每个工具调用按 fastmcp 的默认并发处理；discover_creators_by_hashtag 同步阻塞直到结果返回，limit=100 大概 10–30 秒
  • IG/FB 走住宅代理池，每个账号每天合理上限 ~200 次搜索，避免触发风控
  • runs/datasets 在 NAS 内存里保留 1 小时 后 GC —— 拿到结果要及时落库
  • API key 泄露 → 立刻去看板撤销，重新生成
